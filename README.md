# Zomato vs Swiggy Hyperlocal Demand Analysis Bengaluru

## Project Summary :

This project focuses on analyzing restaurant data from Zomato and Swiggy to understand customer preferences, demand distribution, and pricing patterns across Bengaluru. The goal was to build a complete end to end data analyst workflow starting from raw data collection to final dashboard creation.

The analysis was performed using structured datasets and involved cleaning, transforming, and integrating data from multiple sources. The project emphasizes practical business understanding by identifying high demand locations, analyzing cost and rating relationships, and comparing platform performance. The final outcome is an interactive dashboard that allows users to explore insights dynamically.

## Problem Statement :

Food delivery platforms operate in highly competitive markets where customer satisfaction, pricing, and location demand play a major role. This project aims to answer key business questions such as which platform performs better, which areas have the highest demand, and how pricing affects customer ratings.

## Objectives :

The primary objective was to analyze and compare Zomato and Swiggy data to extract meaningful insights. The project also aimed to demonstrate strong data cleaning, analysis, and visualization skills using industry relevant tools.

## Dataset Information :

The project uses publicly available datasets from Kaggle.
The Zomato dataset contains information about restaurants including name, location, cuisines, ratings, and cost.
The Swiggy dataset includes restaurant details such as name, city, rating, cost, and cuisine.

The datasets were filtered and standardized to focus only on Bengaluru data and to ensure consistency between both platforms.

## Data Cleaning and Preprocessing :

The raw datasets contained inconsistencies such as missing values, duplicate records, mixed formats, and irrelevant columns. These issues were handled using Excel and Python.

Ratings were cleaned and converted into numeric format. Missing values in rating and votes were handled using appropriate techniques such as imputation and default values. Cost values were standardized by removing currency symbols and converting them into numeric format. Duplicate rows were removed to maintain data quality.

New features such as cost category, demand score, and platform labels were created to support deeper analysis.

## Exploratory Data Analysis :

Exploratory analysis was performed using Python libraries to understand data distribution and patterns. Visualizations such as bar charts, distribution plots, and category comparisons were created to analyze ratings, cost ranges, and cuisine popularity.

The analysis helped identify trends such as the distribution of restaurants across locations, variation in ratings, and dominance of certain cuisines.

## SQL Analysis :

Structured Query Language was used to perform advanced data analysis. Queries were written to calculate average ratings, count restaurants by location, identify top performing cuisines, and compare platform level performance.

Advanced concepts such as group by, filtering, ranking, and common table expressions were used to derive meaningful insights from the data.

## Excel Analysis :

Excel was used to perform initial data cleaning and create pivot based analysis. Pivot tables were used to summarize restaurant counts, ratings, and cost distributions. Charts were created to visualize platform comparison, location demand, and cost segmentation.

An Excel dashboard was also created to present key metrics and insights in a structured format.

## Power BI Dashboard :

An interactive dashboard was built using Power BI to present the final analysis. The dashboard includes key performance indicators such as total restaurants, average rating, and average cost.

Visualizations include platform comparison charts, location demand analysis, cost distribution, and heatmap for rating patterns. Slicers were added to enable dynamic filtering based on platform, location, and cost category.

The dashboard provides a clear and interactive way to explore the data and supports decision making.

## Key Insights :

Zomato shows slightly higher average ratings compared to Swiggy, indicating better customer satisfaction in the dataset.
Certain locations such as BTM, Indiranagar, and Whitefield have a higher concentration of restaurants, indicating strong demand.
Mid range restaurants tend to have more stable and consistent ratings compared to low and high cost segments.
Popular cuisines such as North Indian and Chinese dominate across both platforms.
High demand locations also show higher competition, which impacts rating distribution.

## Tools and Technologies Used :

SQL was used for querying and performing structured analysis.
Excel was used for initial cleaning, pivot tables, and dashboard creation.
Power BI was used to build interactive dashboards and visualize insights.
